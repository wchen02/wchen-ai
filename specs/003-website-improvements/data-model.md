# Data Model: Website Improvements (003)

**Branch**: `003-website-improvements` | **Date**: 2026-02-28

Entities and validation rules derived from the feature spec and existing codebase. No new databases; all data is file-based or build-time generated.

---

## Existing entities (unchanged shape, extended use)

### Writing (frontmatter + computed)

- **Source**: `content/writing/*.mdx`, validated by `WritingSchema` in `src/lib/schemas.ts`
- **Fields**: title, publishDate, updatedAt?, theme, tags[], featured, draft
- **Computed at build**: slug, excerpt, readingTimeMinutes
- **New for 003**: Optional `ogImage` (URL string) in frontmatter for article-specific preview image; optional theme descriptor resolved from config (see below)

### Project (frontmatter + computed)

- **Source**: `content/projects/*.mdx`, validated by `ProjectSchema`
- **Fields**: title, date, status, type[], motivation, problemAddressed, learnings?, url?, github?, featured
- **Computed**: slug
- **New for 003**: Used for filtering/grouping by `type` and `status` (already in schema); no schema change required

---

## New or extended config entities

### Metadata defaults (single source of truth)

- **Source**: Prefer a dedicated `src/lib/metadata-defaults.ts`; alternatively extend `src/lib/site-config.ts`. Use one approach consistently across setup and wiring (tasks T001, T003, T036).
- **Fields**:
  - `siteName`: string (e.g. "Wilson Chen")
  - `defaultOgImageUrl`: string (absolute URL to default preview image)
  - `locale`: string (e.g. "en_US") for og:locale
  - `canonicalBaseUrl`: string (e.g. SITE_URL, no trailing slash)
- **Validation**: All required; used by layout and all page metadata. Reuse `SITE_URL` from site-config for canonical base.

### Theme descriptor

- **Source**: Config (e.g. `src/lib/theme-config.ts` or key-value in site-config)
- **Shape**: Map from theme key (string, matching `Writing.theme`) to short description (string)
- **Usage**: Writing index section labels; optional aria-label or caption for theme navigation; "Read Next" or search can surface theme name + descriptor
- **Validation**: Theme keys used in writings should exist in config; build or lint can warn on missing keys

### Credibility signal (content only)

- **Source**: Copy in homepage hero (e.g. in `src/app/page.tsx` or a small content fragment). Not a structured entity; one or more short strings (e.g. "Previously at X", "Building Y").
- **Validation**: None beyond presence of at least one such line (FR-001).

## Build-time generated (no persistence)

### Search index (optional)

- **Source**: Generated by script (e.g. `scripts/generate-search-index.ts`) from `getWritings()`
- **Output**: JSON file (e.g. `public/search-index.json`) or inlined in a page/layout
- **Shape**:
  - `writings`: Array<{ slug: string, title: string, theme: string, tags: string[] }>
  - Optionally: `themes`: string[] for quick filters
- **Validation**: Zod schema for the index shape; script runs in prebuild or as part of build

### Performance targets (documentation / CI)

- **Source**: Documented in plan and optionally in a small config (e.g. thresholds in CI or a constants file)
- **Fields**: LCP threshold (e.g. 2500 ms), CLS threshold (e.g. 0.1)
- **Usage**: Lighthouse or Web Vitals CI step compares against these; no runtime entity

---

## State transitions

- **Theme config**: Add/remove theme key and descriptor; no state machine.
- **Metadata defaults**: Change site name or default image URL; all pages consuming the module pick up the change on next build.
- **Writings/Projects**: Add or edit MDX; build recalculates excerpts, related posts, and optional search index. Draft flag already controls inclusion (existing behavior).

---

## Validation rules (from spec)

- **FR-004**: Every theme used in writing frontmatter SHOULD have a descriptor in theme config (warn if missing).
- **FR-020**: Default metadata (site name, default OG image, locale) MUST be defined in one module and imported everywhere.
- **FR-021**: Internal links (writing/project slugs) validated at build; broken links fail build or emit clear warning (existing `validate-links.ts` extended or kept as-is).
- **Search index**: If generated, schema-validate the output before writing to disk.
